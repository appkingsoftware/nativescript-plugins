Quickstart

If you want a quickstart, clone the repo and run the demo app. There are both vanilla NativeScript, and Angular NativeScript demo apps.

Usage

NOTE: Only works with NativeScript v7+

ns plugin add @rob4226/nativescript-admob

iOS

‚ö†Ô∏è Important! This is required or your app will crash on start-up! ‚ö†Ô∏è

Open your App_Resources/iOS/Info.plist file and add this (replace the value by the actual AdMob app ID of your app!):

  <key>GADApplicationIdentifier</key>
  <string>ca-app-pub-9517346003011652~2508636525</string>
  <key>SKAdNetworkItems</key>
  <array>
    <dict>
      <key>SKAdNetworkIdentifier</key>
      <string>cstr6suwn9.skadnetwork</string>
    </dict>
  </array>

NOTE cstr6suwn9.skadnetwork is Google's SKAdNetwork identifier but there are many other 3rd party SKAdNetwork identifiers provided to Google that you can also add. Please see the following link for other SKAdNetwork identifiers:
https://developers.google.com/admob/ios/quick-start#update_your_infoplist
https://developers.google.com/admob/ios/3p-skadnetworks

Also, run pod repo update from a Terminal, otherwise the required Pod version may not be available on your system.

API

NOTE The second parameter in each ad method, callbacks is optional. You may specify all, none, or as many or few as you would like. They allow you to hook into the various events of each ad's lifecycle but are completely optional.

BANNER ADS

Banner ads occupy a spot within an app's layout, either at the top or bottom of the device screen. They stay on screen while users are interacting with the app, and can refresh automatically after a certain period of time. If you're new to mobile advertising, they're a great place to start.

There are various sizes of banner ads available. See the AD_SIZE enum, depicted in the table below, and use it to specify the size property in the create banner options.

AD_SIZE Enum Key Banner Type Description (size in dp WxH)
BANNER Regular Banner 320x50 Available for phones and tablets.
ADAPTIVE_BANNER Adaptive banner Full width of screen (recommended), or adaptiveBannerWidth if specified. Height depends on device, but is is never larger than 15% of the device's height, and never smaller than 50 dp. Designed to be a drop-in replacement for the industry standard 320x50 banner size, as well as the smart banner format they supersede.
LARGE_BANNER Large Banner 320x100 Available for phones and tablets.
MEDIUM_RECTANGLE IAB Medium Rectangle 300x250 Available for phones and tablets.
FULL_BANNER IAB Full-Size Banner 468x60 Available for tablets.
LEADERBOARD IAB Leaderboard 728x90 Available for tablets.

createBanner

 
import { AD_SIZE, NativescriptAdmob } from '@rob4226/nativescript-admob';
import { isIOS } from "@nativescript/core";

//  If you set the first parameter of the plugin constructor, `isProduction` to `false` (the default), then your device will automatically be registered as a test device and show test ads that are safe for testing in development. Set to `true` to show real production ads.
const admob = new NativescriptAdmob(global.isProduction);

admob.createBanner({
  // If this 'view' property is not set, the banner is overlaid on the current top most view
  // view: ..,
  size: AD_SIZE.ADAPTIVE_BANNER,
  // If you don't specify an ad unit id, a testing one will automatically be used.
  iosAdUnitIdId: 'ca-app-pub-9517346003011652/3985369721',
  androidAdUnitId: 'ca-app-pub-3940256099942544/6300978111',
  margins: {
    // If both are set, top wins
    // top: 10
    bottom: 0,
  },
  keywords: ['foo', 'bar'],
  // Optional and only for ADAPTIVE_BANNER. If not specified will use the full width of the screen.
  // It is recommended to to let an adaptive banner take up the full width screen so it's better to not use this option.
  // adaptiveBannerWidth: 300,
},
{
  // Optional callbacks:
  onAdClicked: () => console.log('Banner ad "onAdClicked" callback called.'),
  onAdClosed: () => console.log('Banner ad "onAdClosed" callback called.'),
  onAdOpened: () => console.log('Banner ad "onAdOpened" callback called.'),
  onAdFailedToLoad: (error) => console.log(`Banner ad "onAdFailedToLoad" callback called...error: ${error}`),
  onAdImpression: () => console.log('Banner ad "onAdImpression" callback called.'),
  onAdLoaded: () => console.log('Banner ad "onAdLoaded" callback called.'),
},
)
.then(
  () => (this.message = 'Banner created'),
  (error) => (this.message = 'Error creating banner: ' + error),
);

hideBanner

NOTE: If you want to show a different banner than the one showing you don't need to call hideBanner since createBanner will do that for you to prevent your app from crashing.

admob.hideBanner().then(
  () => console.log("admob hideBanner done"),
  (error) => console.log("admob hideBanner error: " + error)
);

// The call backs are optional so you can just do:
admob.hideBanner().then();

INTERSTITIAL AD

Interstitial ads are full-screen ads that cover the interface of their host app. They're typically displayed at natural transition points in the flow of an app, such as between activities or during the pause between levels in a game. When an app shows an interstitial ad, the user has the choice to either tap on the ad and continue to its destination or close it and return to the app.

Note that interstitial ads need to be loaded before they can be shown. Use preloadInterstitial, then at any time after its Promise resolves, call showInterstitial. This will hide the preloading delay for your users.

preloadInterstitial

Use this, for instance, while loading your view, so it's ready for the moment you want to actually show it (by calling showInterstitial).

admob.preloadInterstitial({
  // If you don't specify an ad unit id, a testing one will automatically be used.
  iosAdUnitId: 'ca-app-pub-9517346003011652/6938836122',
  androidAdUnitId: 'ca-app-pub-9517346003011652/6938836122',
  keywords: ['foo', 'bar'],
},
{
  // Optional callbacks:
  onAdDismissedFullScreenContent: () => console.log('Interstitial ad "onAdDismissedFullScreenContent" callback called.'),
  onAdFailedToShowFullScreenContent: (error) => console.log(`Interstitial ad "onAdFailedToShowFullScreenContent" callback called...error: ${error}`),
  onAdImpression: () => console.log('Interstitial ad "onAdImpression" callback called.'),
  onAdShowedFullScreenContent: () => console.log('Interstitial ad "onAdShowedFullScreenContent" callback called.'),
  onIOSAdClicked: () => console.log('Interstitial ad "onIOSAdClicked" callback called (only called on iOS devices).'),
},
)
.then(
  () => (this.message = 'Interstitial ad preloaded'),
  (error) => (this.message = 'Error preloading interstitial ad: ' + error),
);

showInterstitial

At any time after preloadInterstitial successfully resolves, you can call showInterstitial.

Note, that when you want to call showInterstitial again, you must also call preloadInterstitial again, because those ads can't be reused.

admob.showInterstitial().then(
  () => {
    // this will resolve almost immediately, and the interstitial is shown without a delay because it was already loaded.
    console.log("Interstitial ad showing.");
  },
  (error) => {
    console.log("Admob showInterstitial error: " + error);
  }
)

REWARDED AD

Ads that reward users for watching short videos and interacting with playable ads and surveys. Used for monetizing free-to-play apps.

Note that just like interstitial ads, rewarded ads need to be loaded before they can be shown. Use preloadRewardedVideoAd, then at any time after its Promise resolves, call showRewardedVideoAd. This will hide the preloading delay for your users.

preloadRewardedVideoAd

Use this for instance while loading your view, so it's ready for the moment you want to actually show it (by calling showRewardedVideoAd).

admob.preloadRewardedVideoAd({
    // If you don't specify an ad unit id, a testing one will   automatically be used.
    iosAdUnitId: 'ca-app-pub-3940256099942544/1712485313',
    androidAdUnitId: 'ca-app-pub-3940256099942544/5224354917',
    keywords: ['foo', 'bar'],
  },
  {
    // Optional callbacks:
    onAdDismissedFullScreenContent: () =>
      console.log('Rewarded ad "onAdDismissedFullScreenContent" callback called.'),
    onAdFailedToShowFullScreenContent: (error) => console.log(`Rewarded ad "onAdFailedToShowFullScreenContent" callback called...error: ${error}`),
    onAdImpression: () => console.log('Rewarded ad "onAdImpression" callback called.'),
    onAdShowedFullScreenContent: () => console.log('Rewarded ad "onAdShowedFullScreenContent" callback called.'),
    onIOSAdClicked: () => console.log('Rewarded ad "onIOSAdClicked" callback called (only called on iOS devices).'),
  },
)
.then(
  () => (this.message = 'Rewarded video ad preloaded'),
  (error) =>
    (this.message = 'Error preloading rewarded video ad: ' + error),
);

showRewardedVideoAd

At any time after preloadRewardedVideoAd successfully resolves, you can call showRewardedVideoAd.

The promise resolves with an AdReward object, which contains the user's reward for watching the rewarded ad.

Note that when you want to call showRewardedVideoAd again, you must also call preloadRewardedVideoAd again because those ads can't be reused.

‚ö†Ô∏è Before displaying a rewarded ad to users, you must present the user with an explicit choice to view rewarded ad content in exchange for a reward. Rewarded ads must always be an opt-in experience. See the Admob documents for more information.

admob.showRewardedVideoAd()
  .then(
    (reward) => {
      this.message = `You have earned a reward! Type: ${reward.type} Amount: ${reward.amount}`;
    })
  .catch(
    (error) => (this.message = 'Error showing rewarded video ad: ' + error)
  );
