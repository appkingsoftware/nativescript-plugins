Interface InAppPurchaseStateUpdateListener

Passed to the constructor of the NativescriptPurchase plugin. These callbacks are where a lot of the action happens. They are called during various points of the in-app purchase lifecycle.

onUpdateHistory

  • onUpdateHistory(purchaseTransactionState: InAppPurchaseTransactionState): void
    • Defined in packages/nativescript-purchase/index.d.ts:172

    Listener to get notified when restoring the users already purchased items. Receives updates when purchaseHistory is called.

    remarks

    On Android, returns the most recent purchase made by the user for each SKU, even if that purchase is expired, canceled, or consumed.

    remarks

    On iOS, calls restoreCompletedTransactions() to restore finished transactions, and calls this callback for each transaction with a transaction state of SKPaymentTransactionState.Restored.
